# Healthcare AI — Trending This Week (2026-07-02)

## Hot on GitHub

### MedAgentGym (NEW — June 2026)
- **Repo**: [gersteinlab/MedAgentGym](https://github.com/gersteinlab/MedAgentGym)
- **Why trending**: Just published to arXiv (2506.04405). Provides a scalable environment for training LLM agents on biomedical data science tasks with code execution. First gym-style environment specifically for clinical data agents.
- **Signal**: ArXiv → GitHub to production pipeline is 3-6 months. Worth tracking now.

### Awesome-AI-Agents-for-Healthcare
- **Repo**: [AgenticHealthAI/Awesome-AI-Agents-for-Healthcare](https://github.com/AgenticHealthAI/Awesome-AI-Agents-for-Healthcare)
- **Why trending**: Rapid community curation of 2025-2026 agent papers. Covers: diagnostic agents, radiology benchmarks (ABRA), clinical prediction (AgentRx), and 3D CT benchmarking (DeepTumorVQA).
- **Signal**: Go-to reference for sales decks and capability assessments.

### MedMemoryBench
- **What**: New benchmark for evaluating agent memory in personalized healthcare settings
- **Why it matters**: Points to a key gap — agents that remember patient history across sessions. First formal benchmark for this capability.
- **Implications for Globant**: Persistent patient context is a design requirement for any longitudinal care agent.

- 62.6% of US hospitals connected to Epic now use ambient AI documentation tools (mid-2026)

- **Globant opportunity**: Implementation, customization, and downstream analytics on top of ambient doc infrastructure

- EHRAgent and KG4Diagnosis (knowledge graph + LLM) showing strong results on clinical Q&A
- BCG (June 2026) forecasts AI agents become primary workflow interface in 3-5 years
- Key gap: integration with live EHR APIs (FHIR R4) — strong Globant services play

- Watch: Meditron + clinical fine-tuning services = recurring revenue opportunity
